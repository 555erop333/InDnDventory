# InDnDventory

Веб-приложение для управления инвентарем персонажей D&D 5e с поддержкой мультиплеера в реальном времени. Мастер (DM) и игроки работают в рамках кампаний, каждый со своими персонажами, предметами и валютой.

## Возможности

**Кампании и пользователи**
- Регистрация и авторизация (JWT)
- Создание кампаний (роль Мастера) и присоединение по коду приглашения (роль Игрока)
- Мастер видит всех персонажей в кампании, игрок — только своих
- Экспорт и импорт кампаний целиком (персонажи + квесты)
- Удаление кампаний и исключение игроков (только DM)

**Инвентарь**
- Управление предметами: создание, редактирование, удаление
- Категории: Оружие, Броня, Зелья, Свитки, Снаряжение, Сокровища, Разное
- Редкость: от Обычного до Артефакта
- Свойства: вес, стоимость, магический, требует настройки, изображение
- Пометка предметов как "На продажу" и массовая продажа хлама
- Поиск и фильтрация по категориям

**Валюта и нагрузка**
- Кошелек с золотыми монетами и быстрыми действиями (+1, +10, +50, +100 и обратно)
- Расчет общего веса с учетом Силы персонажа (нагрузка / перегруз по правилам D&D 5e)

**Квесты и рецепты**
- Мастер создает квесты и крафтовые рецепты
- Назначение квестов конкретным персонажам или всей партии

**Чат**
- Личные сообщения между участниками кампании
- DM управляет правами переписки между игроками и видит все сообщения

**Статусы персонажей**
- DM может назначать статусы и эффекты персонажам

**Синхронизация**
- Обновления в реальном времени через WebSocket (Socket.IO)
- Уведомления об изменениях инвентаря других игроков (для DM)
- Автосохранение с debounce

**История**
- Лог всех действий: добавление, удаление, изменение предметов и валюты
- DM видит глобальный лог по всем персонажам

## Стек технологий

| Компонент | Технология |
|-----------|------------|
| Фронтенд | React 19, TypeScript, Vite, Tailwind CSS, Lucide Icons |
| Бэкенд | Node.js, Express 4, Socket.IO |
| БД | MariaDB (MySQL-совместимая) |
| Аутентификация | JWT + bcrypt |
| Деплой | Docker Compose, Nginx (reverse proxy + SSL) |

## Запуск

### Docker (рекомендуется)

1. Склонируйте репозиторий
2. Настройте SSL-сертификаты в папке `certs/` (`fullcert.pem` и `key.pem`)
3. Измените пароли в `docker-compose.yml`
4. Запустите:

```bash
docker compose up -d --build
```

Приложение будет доступно на портах 80 (HTTP → редирект) и 443 (HTTPS).

### Локальная разработка

**Фронтенд:**
```bash
npm install
npm run dev
```

**Бэкенд:**
```bash
cd api
npm install
npm start
```

Требуется запущенный MariaDB/MySQL с базой `dnd_inventory`. Таблицы создаются автоматически при первом запуске.

## Переменные окружения (API)

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `DB_HOST` | Хост базы данных | `db` |
| `DB_USER` | Пользователь БД | `dnd_user` |
| `DB_PASS` | Пароль БД | `dnd_user_password` |
| `JWT_SECRET` | Секрет для подписи токенов | встроенный (сменить!) |
